## Техническое распределение функций в зависимости от уровня прокси

### Access

1. Отвечает за реализацию аутентификации: доступен по TCP на своем порту всем, но впускает запросы только по логину и паролю/
2. Отвечает за авторизацию запросов к API - на основе по запрошенного URL или домена/
3. Отвечает за журналирование/
4. Отвечает за кеширование там, где это возможно. Для TLS CONNECT это не работает из-за bypass слепоты.
5. Отвечает за прозрачную отказоустойчивость как публичного эндпоита сервиса прокси для пользователей, так и за автоматическое переключение между upstream прокси.
6. Развертывается парой у одного большого и белого ISP в едином L2 домене (необходимо для VRRP).

Конфигурации:
- Squid: [access/squid.conf](access/squid.conf)
- Keepalived: [access/keepalived.conf](access/keepalived.conf)

### Upstream

1. Отвечает за анонимный транспорт с узким ACL, пропуская запросы только с IP прокси из access.
2. Строгий ACL по TCP: подключения к себе на порт прокси пропускает фаерволлом только с адресов access прокси.
3. Автоматически обновляет пакетную базу ОС без участия админа.
4. Не ведет журнала, не кеширует.
5. Развертывается минимум на 2 любых серверах у двух разных ISP вне РФ для отказоустойчивости.

Конфигурация: [upstream/squid.conf](upstream/squid.conf)
