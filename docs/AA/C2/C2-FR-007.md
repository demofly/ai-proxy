# C2-FR-007 — Конфигурации клиентов

## Ссылки

- Родительские требования C1: [C1-BC-002](../C1/C1-BC-002.md)
- Дочерние требования C3: [C3-AD-001](../C3/C3-AD-001.md)

## Описание

Система генерирует готовые конфигурационные файлы для клиентских приложений, обеспечивая простоту подключения инженеров к прокси-сервису. Конфигурации адаптированы под конкретные инструменты и содержат корректные параметры подключения (адрес прокси, порт, credentials).

### Типы клиентских конфигураций

#### 1. VSCode settings.json (для Claude и Gemini)

Генерируются файлы `settings.json` для расширений VSCode:
- **Claude (Anthropic):** настройка HTTP proxy для Claude extension
- **Gemini (Google):** настройка HTTP proxy для Gemini extension

Содержимое:
```json
{
  "http.proxy": "http://user:password@access-proxy-vip:port",
  "http.proxyStrictSSL": false
}
```

#### 2. FoxyProxy JSON (для браузеров)

Генерируется конфигурация FoxyProxy с 14 regex-паттернами для автоматического проксирования запросов к AI-сервисам:

| # | Паттерн | Целевой сервис |
|---|---|---|
| 1 | `*anthropic.com*` | Claude |
| 2 | `*openai.com*` | ChatGPT |
| 3 | `*chatgpt.com*` | ChatGPT |
| 4 | `*googleapis.com*` | Gemini |
| 5 | `*aistudio.google.com*` | Gemini |
| 6 | `*github.com*` | GitHub Copilot |
| 7 | `*copilot*` | GitHub Copilot |
| 8 | `*cursor.sh*` | Cursor |
| 9–14 | Прочие AI-сервисы и зависимости | npm, pip, etc. |

#### 3. PowerShell-скрипты

Скрипты для установки переменных окружения прокси:
- `set-proxy.ps1` — устанавливает `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`
- `unset-proxy.ps1` — сбрасывает переменные

```powershell
# AI-GENERATED — NOT REVIEWED: SECTION START
$env:HTTP_PROXY = "http://user:password@access-proxy-vip:port"
$env:HTTPS_PROXY = "http://user:password@access-proxy-vip:port"
$env:NO_PROXY = "localhost,127.0.0.1,.local"
# AI-GENERATED — NOT REVIEWED: SECTION END
```

### Генерация конфигураций

Все клиентские конфигурации генерируются Ansible из Jinja2-шаблонов с использованием переменных:
- `proxy_host` — адрес access-прокси (VIP)
- `proxy_port` — порт прокси
- `proxy_users` — список пользователей и паролей
- `allowed_domains` / `allowed_domain_patterns` — для FoxyProxy-паттернов

## Критерии приёмки

| # | Критерий | Метрика / Способ проверки | Целевое значение |
|---|----------|---------------------------|------------------|
| 1 | VSCode settings.json содержит корректный proxy URL | Парсинг JSON; валидация URL | Валидный URL с credentials |
| 2 | FoxyProxy JSON содержит все 14 паттернов | Подсчёт regex-записей в JSON | 14 паттернов |
| 3 | PowerShell-скрипт устанавливает env vars | Выполнение скрипта; проверка $env:HTTP_PROXY | Переменная установлена |
| 4 | Конфигурации генерируются при развёртывании | ansible-playbook + проверка output | Файлы созданы |

## Доказательство реализации

### Конструктивное

Реализовано через Ansible-задачи:
- Jinja2-шаблоны для каждого типа конфигурации.
- Задача `template:` генерирует файлы в output-директорию.
- Переменные подставляются из inventory и group_vars.

### Аналитическое

**Три типа конфигураций:** покрывают основные рабочие инструменты инженеров — IDE (VSCode), браузер (FoxyProxy), CLI (PowerShell env vars). Это минимизирует время настройки для новых пользователей.

**FoxyProxy с regex:** позволяет автоматически маршрутизировать только AI-трафик через прокси, а остальной — напрямую. Это снижает нагрузку на прокси и повышает производительность.

### Негативное

| Риск | Митигация |
|---|---|
| Credentials в открытом виде в конфигурациях | Конфигурации распространяются по внутренним каналам; доступ только для авторизованных инженеров |
| Паттерны FoxyProxy устаревают при добавлении новых AI-сервисов | Перегенерация через Ansible при обновлении allowed_domains |
| Конфигурации не синхронизированы с изменениями прокси | Ansible генерирует всё из единого источника (inventory) |
