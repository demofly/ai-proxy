# C4-TM-002 -- Шаблон Squid Upstream (squid-upstream.conf.j2)

## Ссылки

- Родительские требования C3: [C3-SU-001](../C3/C3-SU-001.md)
- Связанные C4: [C4-PB-001](C4-PB-001.md) (деплоит шаблон), [C4-TM-001](C4-TM-001.md) (access-аналог), [C4-TM-004](C4-TM-004.md) (firewall для upstream)

## Описание

Файл `Servers/deploy/templates/squid-upstream.conf.j2` (~101 строка) -- Jinja2-шаблон конфигурации Squid для upstream-прокси. Генерирует `/etc/squid/squid.conf` на upstream-узлах. Обеспечивает анонимный транспорт с ограничением доступа по IP (только access-прокси), без аутентификации, без кеширования, без логирования.

## Ключевые механизмы

### Ограничение доступа по IP

ACL генерируется динамически из группы `access_proxies`:

```
{% for host in groups['access_proxies'] %}
acl access_proxy src {{ hostvars[host]['ansible_host'] }}
{% endfor %}
http_access allow access_proxy
http_access deny all
```

Только IP-адреса access-прокси допускаются к upstream. Все остальные запросы отклоняются.

### Анонимный режим

Upstream-прокси удаляет все заголовки, раскрывающие использование прокси:

- `forwarded_for off` -- не добавлять X-Forwarded-For
- `via off` -- не добавлять Via
- Удаление: X-Forwarded-For, Via, X-Real-IP (request), Via, X-Cache (reply)

### Минимальная конфигурация

В отличие от access-прокси ([C4-TM-001](C4-TM-001.md)):

- Нет аутентификации
- Нет domain whitelist
- Нет замены User-Agent
- Нет кеширования (по умолчанию)
- Нет логирования (по умолчанию)
- Нет upstream peers (это и есть конечный upstream)

### Адаптивные ресурсы

- **SMP workers** = vCPU
- **memory_pools_limit** = 20% RAM
- **read_timeout** = переменная (по умолчанию 15 минут, для долгих AI-запросов)

## Критерии приёмки

| # | Критерий | Метрика / Способ проверки | Целевое значение |
|---|----------|---------------------------|------------------|
| 1 | Валидность конфигурации | `squid -k parse -f /etc/squid/squid.conf` | exit code 0 |
| 2 | Доступ только от access-прокси | HTTP-запрос от стороннего IP | 403 Forbidden |
| 3 | Доступ от access-прокси | HTTP-запрос от IP access-прокси | 200 OK |
| 4 | Анонимность | Проверка заголовков на целевом сервере | Нет Via, X-Forwarded-For |
| 5 | Логирование отключено | Проверка `/var/log/squid/` | Нет access.log |

## Доказательство реализации

### Конструктивное

Шаблон реализует минимальную конфигурацию upstream-прокси: доступ только от access-прокси по IP, полная анонимизация, отключение логирования и кеширования. Конфигурация адаптируется к ресурсам хоста (CPU, RAM).

### Трассировочное

| C3 | C4 |
|---|---|
| [C3-SU-001](../C3/C3-SU-001.md) | C4-TM-002 -- конфигурация upstream Squid |

### Эмпирическое

- Валидация при деплое: `validate: 'squid -k parse -f %s'`
- Проверка доступа от access IP: curl через upstream с IP access-прокси
- Проверка блокировки от стороннего IP: curl через upstream с другого IP

### Негативное

- **Несанкционированный доступ** -- ACL по IP, deny all для остальных
- **Утечка метаданных** -- полная анонимизация заголовков
- **Перегрузка памяти** -- ограничение 20% RAM
