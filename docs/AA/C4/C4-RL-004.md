# C4-RL-004 — Роль DNS Resolver (dns_resolver)

## Ссылки

- Родительский C3: [C3-DN-001](../C3/C3-DN-001.md)
- Файл: `Servers/deploy/roles/dns_resolver/`

## Описание

Ansible-роль `dns_resolver` устанавливает и настраивает кеширующий DNS-резолвер Unbound на каждой ноде инфраструктуры. Реализует каскадную архитектуру: upstream-ноды резолвят через публичные DNS, access-ноды пересылают на upstream unbound.

## Структура роли

```
roles/dns_resolver/
├── defaults/main.yml      # Параметры по умолчанию
├── handlers/main.yml      # Handler: Restart unbound
├── meta/main.yml          # Метаданные роли
├── tasks/main.yml         # Основные задачи (141 строка)
└── templates/
    └── unbound.conf.j2    # Шаблон конфигурации Unbound
```

## Параметры (defaults)

| Переменная | Значение | Описание |
|---|---|---|
| `dns_resolver_port` | 53 | Порт unbound |
| `dns_resolver_dnssec` | true | DNSSEC-валидация |
| `dns_resolver_min_ttl` | 300 | Минимальный TTL (5 мин) |
| `dns_resolver_max_ttl` | 86400 | Максимальный TTL (24 ч) |
| `dns_resolver_neg_ttl` | 60 | Negative TTL |
| `dns_resolver_prefetch` | true | Prefetch популярных записей |

### Автовычисляемые параметры

| Параметр | Формула | Описание |
|---|---|---|
| `_dns_threads` | `max(ansible_processor_vcpus, 1)` | Потоки unbound |
| `_dns_cache_mb` | `max(ansible_memtotal_mb / 32, 16)` | Размер кеша (мин 16 MB) |

## Задачи (tasks/main.yml)

| # | Задача | Условие |
|---|--------|---------|
| 1 | Авторасчёт ресурсов (threads, cache) | always |
| 2 | Установка unbound + dns-root-data | Debian: apt, RHEL: dnf |
| 3 | Создание /var/lib/unbound, root.key | always |
| 4 | DNSSEC trust anchor (`unbound-anchor`) | creates |
| 5 | Forwarders: access → upstream unbound IPs | `proxy_role == 'access'` |
| 6 | Forwarders: upstream → публичные DNS | `proxy_role == 'upstream'` |
| 7 | Deploy unbound.conf.j2 (с `unbound-checkconf`) | always |
| 8 | Отключение systemd-resolved stub listener | если active |
| 9 | /etc/resolv.conf → `nameserver 127.0.0.1` | always |
| 10 | Enable + start unbound | always |

## Шаблон (unbound.conf.j2)

### Интерфейсы

| proxy_role | Интерфейсы | Доступ |
|---|---|---|
| upstream | `127.0.0.1`, `ansible_host` | localhost + access proxy IPs |
| access | `127.0.0.1` | только localhost |

### Forwarding

| proxy_role | Forwarders | Протокол |
|---|---|---|
| upstream | 9.9.9.9, 8.8.8.8, 1.1.1.1 | plain DNS (port 53) |
| access | upstream ansible_host IPs | plain DNS (port 53) |

### Безопасность (server section)

- `hide-identity: yes`, `hide-version: yes`
- `harden-glue: yes`, `harden-dnssec-stripped: yes`
- `use-caps-for-id: yes` (0x20 encoding)
- `access-control: 0.0.0.0/0 refuse` (default deny)

## Критерии приёмки

| # | Критерий | Метрика / Способ проверки | Целевое значение |
|---|----------|---------------------------|------------------|
