# C4-RL-001 -- Роль CrowdSec (roles/crowdsec/)

## Описание

Директория `Servers/deploy/roles/crowdsec/` -- Ansible-роль для установки и настройки CrowdSec IPS (Intrusion Prevention System) с firewall bouncer на nftables. CrowdSec обеспечивает коллективную защиту от атак: локальные детекции + TI-фиды от сообщества (70k+ серверов) через CAPI.

## Ключевые механизмы

### Установка коллекций с retry

```yaml
- name: Install CrowdSec collections
  ansible.builtin.command:
    cmd: "cscli collections install {{ item }}"
  loop:
    - crowdsecurity/linux
    - crowdsecurity/nginx
    - crowdsecurity/http-cve
    - crowdsecurity/whitelist-good-actors
  retries: 3
  delay: 10
  until: cscli_install.rc == 0
```

Коллекции:

- **crowdsecurity/linux** -- базовые сценарии обнаружения для Linux (SSH brute-force, сканеры)
- **crowdsecurity/nginx** -- обнаружение атак на nginx (bad user-agent, path traversal, exploits)
- **crowdsecurity/http-cve** -- обнаружение эксплуатации известных CVE
- **crowdsecurity/whitelist-good-actors** -- whitelist легитимных ботов (Googlebot, Bingbot и др.)

### Acquisition config (acquis.yaml.j2)

Шаблон определяет источники логов для мониторинга:

| Источник | Условие | Тип | Описание |
|---|---|---|---|
| `/var/log/squid/access.log` | access + enable_logging | squid | HTTP-атаки, сканеры, brute-force |
| `/var/log/auth.log` | всегда | syslog | SSH brute-force, invalid users |
| journalctl `_SYSTEMD_UNIT=ssh.service` | всегда | syslog | Дублирование SSH через systemd |

Squid-логи мониторятся только на access-прокси с включённым логированием.

### Systemd memory limits

Ограничение памяти через systemd overrides (`MemoryMax`):

- CrowdSec engine: `crowdsec_memory_mb` (по умолчанию 512 MB)
- Firewall bouncer: `crowdsec_fw_bouncer_memory_mb` (по умолчанию 128 MB)

### CAPI (Community API)

CrowdSec автоматически подключается к CAPI при первом запуске. Это обеспечивает:

- Получение TI-фидов от сообщества (70k+ серверов)
- Автоматический бан IP-адресов, замеченных в атаках на другие серверы
- Отправка локальных алертов в сообщество (анонимно)

### Конструктивное

Роль реализует полный цикл установки CrowdSec: от подключения apt-репозитория до запуска сервисов с ограничением памяти. Механизм retry при установке коллекций обеспечивает устойчивость к временным сетевым проблемам. Acquisition-конфигурация адаптируется к роли узла (access/upstream).

- **Broken dpkg** -- `dpkg --configure -a` перед установкой
