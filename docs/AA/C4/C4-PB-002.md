# C4-PB-002 -- Инвентори (inventory.yml)

## Ссылки

- Родительские требования C3: [C3-AD-001](../C3/C3-AD-001.md)
- Связанные C4: [C4-PB-001](C4-PB-001.md) (плейбук, потребитель переменных)

## Описание

Файл `Servers/deploy/inventory.yml` (~135 строк) -- Ansible inventory, определяющий топологию инфраструктуры AI Assistants Proxy: группы хостов, их роли, IP-адреса и полный набор конфигурационных переменных.

## Структура файла

| Секция | Описание |
|---|---|
| **all.children.access_proxies** | Группа access-прокси (2 хоста: access-01, access-02) с per-host переменными Keepalived (state, priority) |
| **access_proxies.vars** | Переменные группы: proxy_role=access, squid_port=443, enable_auth/logging/caching, behind_nginx, Keepalived-настройки |
| **all.children.upstreams** | Группа upstream-прокси (1 хост: upstream01.example.com) |
| **upstreams.vars** | Переменные группы: proxy_role=upstream, squid_port=80, без аутентификации/логирования/кеширования |
| **all.vars** | Глобальные переменные: ansible_user, python interpreter, read_timeout, CrowdSec, MTProxy, DNS, allowed_domains, allowed_domain_patterns |

## Ключевые механизмы

### Двухуровневая иерархия хостов

Инвентори организован в две группы с разными ролями:

- **access_proxies**: клиентские точки входа с аутентификацией, кешированием, Keepalived VIP
- **upstreams**: анонимные транспортные узлы с firewall, MTProxy

Переменная `proxy_role` определяет поведение плейбука для каждой группы.

### Управление секретами

Инвентори содержит чувствительные данные:

- `keepalived_auth_pass` -- пароль VRRP-аутентификации
- `mtproxy_secret` -- hex-секрет MTProxy (Fake-TLS)
- IP-адреса хостов

### Domain whitelist

Два механизма фильтрации доменов:

1. **allowed_domains** -- явный список доменов (dashscope.aliyuncs.com, copilot-proxy.githubusercontent.com и др.)
2. **allowed_domain_patterns** -- regex-паттерны для групп доменов (\.ai$, OpenAI, Anthropic, Google Gemini, GitHub Copilot, Microsoft/Azure, Cursor AI, DeepL)

## Доказательство реализации

### Конструктивное

Инвентори определяет двухуровневую топологию (access + upstream) с чётким разделением ответственности. Переменные разнесены по трём уровням (per-host, group, global), обеспечивая переопределение через стандартный механизм приоритетов Ansible.

### Трассировочное

| C3 | C4 |
|---|---|
| [C3-AD-001](../C3/C3-AD-001.md) | C4-PB-002 -- определение топологии и конфигурации |

### Эмпирическое

- Валидация: `ansible-inventory -i inventory.yml --list`
- Доступность хостов: `ansible all -i inventory.yml -m ping`
- Проверка переменных: `ansible all -i inventory.yml -m debug -a "var=proxy_role"`

### Негативное

- **Чувствительные данные** -- пароли и секреты хранятся в открытом виде (рекомендуется ansible-vault для продакшена)
- **Hardcoded IP** -- IP-адреса зафиксированы; изменение инфраструктуры требует обновления инвентори
- **Keepalived включён** -- `enable_keepalived: true` с VIP 192.168.0.2, VRID 51; MASTER/BACKUP распределение по хостам
