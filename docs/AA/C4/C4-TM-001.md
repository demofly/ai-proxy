# C4-TM-001 -- Шаблон Squid Access (squid-access.conf.j2)

## Ссылки

- Родительские требования C3: [C3-SA-001](../C3/C3-SA-001.md)
- Связанные C4: [C4-PB-001](C4-PB-001.md) (деплоит шаблон), [C4-PB-002](C4-PB-002.md) (источник переменных), [C4-TM-002](C4-TM-002.md) (upstream-аналог)

## Описание

Файл `Servers/deploy/templates/squid-access.conf.j2` (~204 строки) -- Jinja2-шаблон конфигурации Squid для access-прокси. Генерирует `/etc/squid/squid.conf` на access-узлах. Обеспечивает аутентификацию клиентов, domain whitelist, маршрутизацию через upstream peers с балансировкой, анонимизацию запросов и адаптивную настройку производительности.

## Ключевые механизмы

### Аутентификация (условная)

При `enable_auth: true`:

- Программа: `/usr/lib/squid/basic_ncsa_auth` с файлом паролей `/etc/squid/passwd`
- 5 дочерних процессов (startup=5, idle=1)
- TTL кредов: 2 часа, case-sensitive
- ACL `authenticated proxy_auth REQUIRED`

### Domain whitelist (двойной)

Два взаимодополняющих механизма фильтрации:

1. **allowed_domains** -- `acl allowed_domains dstdomain {domain}` для каждого домена из списка
2. **allowed_domain_patterns** -- `acl allowed_domain_regexp dstdom_regex -i {pattern}` для regex-паттернов

Оба ACL используются в правилах доступа: `http_access allow authenticated allowed_domains` и `http_access allow authenticated allowed_domain_regexp`.

### Upstream peers (userhash балансировка)

Для каждого хоста из группы `upstreams`:

```
cache_peer {ip} parent {port} 0 no-query userhash connect-fail-limit=2 connect-timeout=5 name={host}
```

- **userhash** -- sticky sessions по хэшу имени пользователя (автоматический fallback на sourcehash для неаутентифицированных)
- **connect-fail-limit=2** -- после 2 неудач peer помечается мёртвым
- **dead_peer_timeout=15s** -- проверка мёртвых peers каждые 15 секунд
- **never_direct allow all** -- весь трафик через upstream

### Анонимизация и маскировка под браузер

Полная имитация Chrome 138 на Windows 10:

- Удаление: X-Forwarded-For, X-Real-IP, Via, X-Cache, Server
- Замена User-Agent: Chrome/138.0.7204.405
- Добавление: Accept, Accept-Language, Accept-Encoding, Sec-Fetch-*, Sec-Ch-Ua, DNT, Cache-Control

### Адаптивная производительность

- **SMP workers** = количество vCPU (при > 1)
- **memory_pools_limit** = RAM / 5 (20%)
- **cache_mem** = 60% от memory_pools_limit
- **read_timeout** = переменная (по умолчанию 15 минут, настраивается для AI API)

## Критерии приёмки

| # | Критерий | Метрика / Способ проверки | Целевое значение |

## Доказательство реализации

### Конструктивное

Шаблон генерирует полноценную конфигурацию Squid с условными блоками (Jinja2 `{% if %}`) для каждой feature. Механизм upstream-балансировки использует userhash для sticky sessions с автоматическим failover. Анонимизация заменяет заголовки, имитируя реальный браузер Chrome.

### Трассировочное

| C3 | C4 |
|---|---|
| [C3-SA-001](../C3/C3-SA-001.md) | C4-TM-001 -- конфигурация access Squid |

### Эмпирическое

- Валидация конфигурации при деплое: параметр `validate: 'squid -k parse -f %s'` в task
- Ручное тестирование: curl через прокси с/без аутентификации, к разрешённым/запрещённым доменам

### Негативное

- **Отказ upstream** -- connect-fail-limit=2 + dead_peer_timeout=15s обеспечивают быстрое обнаружение и переключение
- **Утечка IP** -- полное удаление proxy-заголовков (X-Forwarded-For, Via, X-Real-IP)
- **DPI-детекция** -- замена User-Agent и добавление полного набора браузерных заголовков
- **Зависание AI API** -- read_timeout настраивается (по умолчанию 15 минут для долгих запросов)
