# DD-Glossary — Словарь данных

## Описание

Словарь данных содержит определения ключевых терминов предметной области проекта AI Assistants Proxy. Термины упорядочены тематически для удобства навигации.

## Термины

### Архитектура прокси

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **Access Proxy** | Прокси-сервер Squid, развёрнутый на access tier в РФ. Точка входа для пользователей. Выполняет аутентификацию, фильтрацию доменов, кеширование и журналирование. Направляет CONNECT-запросы через cache_peer к upstream-прокси. | [C3-SA-001](../C3/C3-SA-001.md) |
| **Upstream Proxy** | Прокси-сервер Squid, развёрнутый на upstream tier вне РФ. Анонимный транспортный уровень без аутентификации и журналирования. Ограничивает доступ по IP-адресам access-прокси (ACL). | [C3-SU-001](../C3/C3-SU-001.md) |
| **cache_peer** | Директива Squid для определения вышестоящего прокси (parent) в цепочке проксирования. На access-прокси настраивается cache_peer к upstream-нодам с балансировкой `userhash`. | [DE-CF-001](DE/DE-CF-001.md) |
| **SMP workers** | Symmetric Multi-Processing workers — механизм параллелизма в Squid 6.x. Позволяет запускать несколько рабочих процессов для обработки запросов на многоядерных серверах. Количество определяется конфигурацией `workers N`. | [DE-CF-001](DE/DE-CF-001.md) |
| **userhash** | Алгоритм балансировки нагрузки в Squid cache_peer. Распределяет запросы между upstream-нодами на основе хеша имени аутентифицированного пользователя, обеспечивая sticky-сессии без хранения состояния. | [DE-CF-001](DE/DE-CF-001.md) |

### Сетевые протоколы и безопасность

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **SNI** | Server Name Indication — расширение TLS, позволяющее клиенту указать имя целевого хоста в открытом виде при установке TLS-соединения. Используется nginx stream для маршрутизации трафика на порту 443 между Squid и MTProxy. | [C3-NX-001](../C3/C3-NX-001.md) |
| **VRRP** | Virtual Router Redundancy Protocol — протокол резервирования маршрутизаторов. Keepalived реализует VRRP для обеспечения отказоустойчивости access-прокси через виртуальный IP (VIP). | [C3-KA-001](../C3/C3-KA-001.md) |
| **VIP** | Virtual IP — виртуальный IP-адрес, управляемый Keepalived через VRRP. При отказе MASTER-ноды VIP автоматически переключается на BACKUP-ноду. Текущий VIP: `1.2.3.4`. | [DE-CF-001](DE/DE-CF-001.md) |
| **ACL** | Access Control List — список управления доступом. В контексте проекта: (1) Squid ACL для фильтрации доменов на access-прокси; (2) Squid ACL по IP-адресам на upstream-прокси; (3) nftables ACL для блокировки IP (CrowdSec). | [DE-AC-001](DE/DE-AC-001.md) |
| **DPI** | Deep Packet Inspection — глубокий анализ сетевых пакетов. Технология, используемая для обнаружения и блокировки протоколов. MTProxy использует fake-TLS для обхода DPI. | [C3-MT-001](../C3/C3-MT-001.md) |
| **fake-TLS** | Техника маскировки протокола MTProxy под стандартный TLS-трафик к произвольному домену (например, example.gov.ru). Используется для обхода DPI-систем, блокирующих протокол MTProto. | [DE-SC-001](DE/DE-SC-001.md) |

### Аутентификация и управление доступом

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **NCSA** | National Center for Supercomputing Applications — формат хранения паролей, используемый Apache httpd и Squid. Файл `passwd` хранит записи в формате `user:hash`. Squid использует хелпер `basic_ncsa_auth` для проверки. | [DE-CR-001](DE/DE-CR-001.md) |
| **htpasswd** | Утилита командной строки Apache для управления файлами паролей в формате NCSA. Поддерживает алгоритмы хеширования: MD5 (`$apr1$`), bcrypt, SHA-1. В проекте используется MD5-crypt. | [DE-CR-001](DE/DE-CR-001.md) |
| **Basic Auth** | Метод аутентификации HTTP, при котором клиент передаёт логин и пароль в заголовке `Proxy-Authorization` в кодировке Base64. Используется на access-прокси для идентификации пользователей. | [DE-CR-001](DE/DE-CR-001.md) |

### MTProxy и Telegram

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **MTProxy** | Прокси-сервер для протокола Telegram MTProto. Реализация: mtg v2 — Go-based MTProxy с поддержкой fake-TLS маскировки. Развёрнут на upstream-нодах. | [C3-MT-001](../C3/C3-MT-001.md) |
| **mtg** | Реализация MTProxy на Go (github.com/9seconds/mtg). Версия 2.x поддерживает fake-TLS, TOML-конфигурацию, ограничение подключений. Конфигурируется через `/etc/mtproxy/mtg.toml`. | [C3-MT-001](../C3/C3-MT-001.md) |
| **MTProxy secret** | Hex-строка, используемая для шифрования соединения между Telegram-клиентом и MTProxy-сервером. Включает закодированное имя fake-TLS домена. Генерируется командой `mtg generate-secret --hex <domain>`. | [DE-SC-001](DE/DE-SC-001.md) |

### IPS и безопасность

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **IPS** | Intrusion Prevention System — система предотвращения вторжений. В проекте: CrowdSec анализирует логи Squid и sshd для обнаружения атак и автоматической блокировки IP через nftables bouncer. | [C3-CS-001](../C3/C3-CS-001.md) |
| **bouncer** | Компонент CrowdSec, применяющий решения о блокировке. Firewall bouncer (`cs-firewall-bouncer`) управляет таблицей `crowdsec-blacklists` в nftables, добавляя и удаляя IP-адреса с таймаутом. | [C3-CS-001](../C3/C3-CS-001.md) |
| **acquisition** | Механизм CrowdSec для чтения источников данных (логов). Для Squid: acquisition настраивается на чтение `/var/log/squid/access.log` с парсером для стандартного формата `squid`. | [DE-LG-001](DE/DE-LG-001.md) |
| **CAPI** | Central API — облачный API CrowdSec для обмена данными об угрозах между участниками сообщества (70k+ серверов). Подключается автоматически при установке CrowdSec. | [C3-CS-001](../C3/C3-CS-001.md) |

### Инфраструктура и развёртывание

| Термин | Определение | Связь с DE/C3 |
|---|---|---|
| **IaC** | Infrastructure as Code — подход к управлению инфраструктурой через код. В проекте: Ansible playbook, roles, Jinja2 templates определяют конфигурацию всех серверов. | [C3-AD-001](../C3/C3-AD-001.md) |
| **Jinja2** | Шаблонизатор Python, используемый Ansible для генерации конфигурационных файлов из шаблонов (.j2) и переменных inventory. Преобразует `inventory.yml` + `*.j2` → целевые конфигурации. | [C3-AD-001](../C3/C3-AD-001.md) |
| **inventory** | Файл инвентаря Ansible (`inventory.yml`), определяющий целевые хосты, группы и переменные. Является единственным источником истины (SSOT) для всех конфигурационных параметров проекта. | [DE-CF-001](DE/DE-CF-001.md) |
| **no_log** | Директива Ansible для подавления вывода задачи в лог. Используется для задач, обрабатывающих секреты (пароли, токены), чтобы исключить их утечку в консоль и файлы логов. | [DE-SC-001](DE/DE-SC-001.md) |
| **logrotate** | Утилита Linux для ротации лог-файлов. Управляет размером и количеством файлов access.log на access-прокси. Конфигурация определяет период ротации, сжатие и максимальное количество архивов. | [DE-LG-001](DE/DE-LG-001.md) |
