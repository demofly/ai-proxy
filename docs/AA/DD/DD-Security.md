# DD-Security — Безопасность данных

## Описание

Документ определяет классификацию конфиденциальности, матрицы доступа и меры защиты для каждой сущности данных проекта AI Assistants Proxy.

## Классификация конфиденциальности

| Сущность | ID | Уровень конфиденциальности | Обоснование | Меры защиты |
|---|---|---|---|---|
| Учётные данные прокси | [DE-CR-001](DE/DE-CR-001.md) | **Высокий** | Компрометация позволяет несанкционированный доступ к прокси; содержит хеши паролей | Хеширование (apr1/MD5-crypt), файловые права 640, `no_log` в Ansible, не в журналах |
| Секреты | [DE-SC-001](DE/DE-SC-001.md) | **Высокий** | Компрометация MTProxy secret позволяет подключиться к MTProxy; Keepalived password позволяет перехватить VIP | Хранение в TOML/inventory, `no_log` в Ansible, не в журналах, ограничение файловых прав 640 |
| Записи журнала доступа | [DE-LG-001](DE/DE-LG-001.md) | **Средний** | Содержат IP-адреса клиентов и имена пользователей; позволяют восстановить историю обращений | Хранение только на access tier, права 640, ротация с TTL, не передаются за пределы хоста |
| Списки управления доступом | [DE-AC-001](DE/DE-AC-001.md) | **Средний** | Раскрывают IP-адреса инфраструктуры (access-прокси) | Генерируются автоматически из inventory, не публикуются |
| Конфигурационные параметры | [DE-CF-001](DE/DE-CF-001.md) | **Низкий-Средний** | Параметры инфраструктуры; часть содержит IP-адреса серверов | VCS (Git) для контроля изменений; чувствительные параметры → отдельные переменные с `no_log` |
| Белый список доменов | [DE-DM-001](DE/DE-DM-001.md) | **Низкий** | Публичные домены AI API, общедоступная информация | Стандартные файловые права 644 |

## Меры защиты по уровням

### Высокий уровень конфиденциальности

| Мера | Описание | Применимость |
|---|---|---|
| Хеширование | Пароли хранятся в виде хеша apr1 (MD5-crypt), не в открытом виде | [DE-CR-001](DE/DE-CR-001.md) |
| Файловые права 640 | Чтение только владельцем (root) и группой (service) | [DE-CR-001](DE/DE-CR-001.md), [DE-SC-001](DE/DE-SC-001.md) |
| Ansible `no_log: true` | Подавление вывода задач с чувствительными переменными | Все задачи с паролями и секретами |
| Исключение из журналов | Секреты не попадают в access.log и системные логи | [DE-SC-001](DE/DE-SC-001.md) |
| Безопасная передача | Логин/пароль передаётся пользователю по безопасному каналу | [DE-CR-001](DE/DE-CR-001.md) |

### Средний уровень конфиденциальности

| Мера | Описание | Применимость |
|---|---|---|
| Ограничение хранения | Данные хранятся только на тех серверах, где необходимы | [DE-LG-001](DE/DE-LG-001.md), [DE-AC-001](DE/DE-AC-001.md) |
| Ротация с TTL | Журналы ротируются и удаляются по расписанию | [DE-LG-001](DE/DE-LG-001.md) |
| Автоматическая генерация | ACL генерируются из inventory, исключая ручные ошибки | [DE-AC-001](DE/DE-AC-001.md) |

### Низкий уровень конфиденциальности

| Мера | Описание | Применимость |
|---|---|---|
| Стандартные файловые права | 644 — чтение для всех, запись только root | [DE-DM-001](DE/DE-DM-001.md) |
| Контроль версий | Изменения отслеживаются через Git | [DE-CF-001](DE/DE-CF-001.md), [DE-DM-001](DE/DE-DM-001.md) |

## Угрозы и контрмеры

| Угроза | Затронутые DE | Вероятность | Влияние | Контрмера |
|---|---|---|---|---|
| Утечка passwd через лог Ansible | DE-CR-001 | Средняя | Высокое | `no_log: true` для задач с паролями |
| Перехват Basic Auth (man-in-the-middle) | DE-CR-001 | Низкая | Высокое | HTTPS CONNECT tunnel (TLS до целевого сервера) |
| Доступ к access.log посторонними | DE-LG-001 | Низкая | Среднее | Файловые права 640, отсутствие SSH для непривилегированных |
| Компрометация MTProxy secret | DE-SC-001 | Низкая | Среднее | Ограничение доступа к mtg.toml, ротация secret |
| Перехват VRRP аутентификации | DE-SC-001 | Низкая | Высокое | Keepalived auth_pass в закрытой сети |
| Раскрытие IP инфраструктуры | DE-AC-001 | Средняя | Среднее | IP не публикуются; VIP — единственный публичный адрес |
