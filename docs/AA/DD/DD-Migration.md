# DD-Migration — Миграция данных

## Описание

Документ описывает стратегию и историю миграций данных проекта AI Assistants Proxy. Проект не использует традиционные СУБД, поэтому миграции данных относятся к изменениям форматов конфигурационных файлов и архитектурным переходам.

## Общая стратегия

- **Нет миграций БД** — проект не использует базы данных
- **Миграции конфигураций** — изменения форматов выполняются через Ansible roles с поддержкой обратной совместимости
- **Паттерн:** Expand-Contract — сначала поддержка обоих форматов, затем удаление устаревшего
- **Откат:** каждая миграция поддерживает rollback через Ansible handlers или отдельные tasks

## Реестр миграций

### MIG-001: mtg.env → mtg.toml (MTProxy v2)

| Параметр | Значение |
|---|---|
| **ID** | MIG-001 |
| **Дата** | Реализовано в текущей версии |
| **Сущность** | [DE-SC-001](DE/DE-SC-001.md), [DE-CF-001](DE/DE-CF-001.md) |
| **Компонент** | [C3-MT-001](../C3/C3-MT-001.md) — MTProxy (mtg) |
| **Причина** | Переход mtg v1 (env-файл) → mtg v2 (TOML-конфигурация) |
| **Обратимость** | Нет (mtg v2 не поддерживает env-формат) |
| **Статус** | Завершена |

**Детали миграции:**

| Параметр | Старый формат (mtg.env) | Новый формат (mtg.toml) |
|---|---|---|
| Секрет | `MTPROTO_SECRET=<hex>` | `secret = "<hex>"` |
| Привязка | `MTPROTO_BIND=:443` | `bind-to = "127.0.0.1:2083"` |
| Формат файла | Environment variables (.env) | TOML key-value pairs |
| Путь | `/etc/mtproxy/mtg.env` | `/etc/mtproxy/mtg.toml` |

**Процедура:** Ansible role `mtproxy` автоматически:
1. Останавливает старый сервис mtg (если запущен)
2. Удаляет старый конфигурационный файл mtg.env
3. Генерирует новый mtg.toml из Jinja2-шаблона
4. Перезапускает сервис

### MIG-002: Squid direct :443 → nginx:443 + Squid:3128

| Параметр | Значение |
|---|---|
| **ID** | MIG-002 |
| **Дата** | Реализовано в текущей версии |
| **Сущность** | [DE-CF-001](DE/DE-CF-001.md) |
| **Компонент** | [C3-NX-001](../C3/C3-NX-001.md) — nginx SNI Router, [C3-SA-001](../C3/C3-SA-001.md) — Squid Access Proxy |
| **Причина** | Совместное использование порта 443 с MTProxy (SNI routing) |
| **Обратимость** | Да (установить `behind_nginx: false` в inventory) |
| **Статус** | Завершена |

**Детали миграции:**

| Параметр | До миграции | После миграции |
|---|---|---|
| Squid listen | `0.0.0.0:443` | `127.0.0.1:3128` |
| nginx | Не установлен | stream SNI router на `0.0.0.0:443` |
| MTProxy доступ | Недоступен на access tier | Через nginx SNI routing |
| Переключатель | — | `behind_nginx: true/false` в inventory |

**Процедура отката:** установить `behind_nginx: false` в inventory.yml → запустить playbook → Squid вернётся на прямое прослушивание :443, nginx будет удалён/остановлен.
