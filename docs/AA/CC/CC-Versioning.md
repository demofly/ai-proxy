# CC-Versioning — Политика версионирования артефактов

## Схемы версионирования

| Схема | Формат | Применение |
|---|---|---|
| **CalVer** | `YYYY-MM-DD` | Документация, архитектурные артефакты |
| **SemVer** | `MAJOR.MINOR.PATCH` | Внешние компоненты с собственным версионированием |
| **Git SHA** | 40-символьный хеш коммита | IaC-артефакты (Ansible playbook, templates, roles) |
| **OS Package** | Версия из пакетного менеджера | Компоненты, устанавливаемые через APT/репозиторий |

## Реестр версий артефактов

| Артефакт | Схема версионирования | Текущая версия | Источник / Файл | Примечание |
|---|---|---|---|---|
| Документация `docs/AA/` | CalVer | 2026-02-23 | `docs/AA/` | Привязана к дате коммита в git |
| Ansible playbook | Git SHA | HEAD | `Servers/deploy/playbook.yml` | Семантическое версионирование не применяется; версия = коммит |
| mtg binary (MTProxy) | SemVer | 2.1.8 | `mtproxy_version` в inventory | Фиксируется в `inventory.yml` |
| Squid | OS Package | Latest from APT | `apt install squid` | Версия определяется пакетным менеджером ОС |
| CrowdSec | OS Package | Latest from repo | `apt install crowdsec` | Версия определяется официальным репозиторием CrowdSec |
| nginx | OS Package | Latest from APT | `apt install nginx` | Версия определяется пакетным менеджером ОС |
| Keepalived | OS Package | Latest from APT | `apt install keepalived` | Версия определяется пакетным менеджером ОС |
| nftables | OS Package | Latest from APT | `apt install nftables` | Версия определяется пакетным менеджером ОС |

## Политика версионирования

### Документация

- Версия документации привязана к дате коммита в git (CalVer: `YYYY-MM-DD`).
- Каждое значимое изменение документации фиксируется в [CC-Changes.md](CC-Changes.md).
- Версионирование API: **N/A** — проект не предоставляет API; является инфраструктурным прокси-сервисом.

### IaC-артефакты (Ansible)

- Версия Ansible playbook, ролей и шаблонов определяется хешем коммита git (Git SHA).
- Семантическое версионирование не применяется: playbook не является библиотекой и не публикуется.
- Для отката к предыдущей версии используется `git checkout <SHA>`.

### Внешние компоненты

- **mtg (MTProxy):** версия фиксируется явно в переменной `mtproxy_version` в inventory. Обновление — ручное, с тестированием.
- **OS-пакеты (Squid, CrowdSec, nginx, Keepalived, nftables):** устанавливаются из официальных репозиториев. Версия определяется состоянием репозитория на момент развёртывания. Для production рекомендуется фиксация через APT pinning.

### Правила обновления

1. **Минорные обновления** OS-пакетов — допускаются без ADR.
2. **Мажорные обновления** (смена major-версии компонента) — требуют ADR и тестирования на staging.
3. **Обновление mtg** — требует проверки совместимости с текущими клиентами MTProxy.
