# CC-RACI — Матрица ответственности

## Роли

| Сокращение | Роль | Описание |
|---|---|---|
| **PO** | Владелец продукта | Определяет требования, приоритеты, принимает архитектурные решения |
| **DO** | DevOps-инженер | Развёртывание, конфигурирование, мониторинг инфраструктуры |
| **DEV** | Инженер-разработчик | Разработка IaC (Ansible), шаблоны, скрипты автоматизации |
| **SEC** | Служба ИБ | Контроль безопасности, ревью политик, реагирование на инциденты |

## Легенда RACI

| Код | Значение | Описание |
|---|---|---|
| **R** | Responsible | Исполняет работу |
| **A** | Accountable | Несёт ответственность за результат, принимает решение |
| **C** | Consulted | Консультируется (двусторонняя коммуникация) |
| **I** | Informed | Информируется (односторонняя коммуникация) |

## Матрица RACI

| Активность | PO | DO | DEV | SEC |
|---|---|---|---|---|
| Определение требований доступа к AI API | **A** | R | C | C |
| Развёртывание инфраструктуры | I | **R/A** | I | C |
| Управление пользователями (htpasswd) | **A** | R | I | C |
| Мониторинг и реагирование на инциденты | I | R | I | **A** |
| Обновление белого списка доменов | **A** | R | C | C |
| Управление CrowdSec (правила, whitelist) | I | R | I | **A** |
| Настройка клиентского ПО | I | C | **R** | I |
| Управление co-deployment (nginx) | I | **R/A** | I | C |
| Ревью безопасности | I | C | I | **R/A** |
| Принятие архитектурных решений | **A** | R | C | C |

## Правила применения

1. Каждая активность имеет ровно одну роль **A** (Accountable). Роль R/A означает совмещение ответственности и исполнения.
2. Минимум одна роль **R** (Responsible) на каждую активность.
3. Количество **C** (Consulted) минимизировано — только роли, чей вклад необходим для качества решения.
4. Матрица обновляется при изменении состава ролей или добавлении новых активностей.
