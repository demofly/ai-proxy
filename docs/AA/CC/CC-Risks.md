## Политика управления рисками

1. **Ревью рисков** — при каждом изменении архитектуры или добавлении нового компонента.
2. **Critical без митигации** — блокер для прохождения Gate.
3. **Новые риски** — регистрируются при обнаружении в ходе разработки или актуализации.
4. **Статусы:** Open (выявлен, митигация не определена), Mitigated (митигация реализована), Accepted (принят без митигации с обоснованием), Closed (риск устранён).
